# Contributions Directory

This directory collects improvements from real projects that should be folded back into the framework.

## Two types of files live here

### 1. Field Notes (`field-notes-{project}-{date}.md`)

Captured **during** project work — quick notes about friction, discoveries, and ideas. These are raw and unprocessed. Anyone working on a project can dump findings here at any time.

Format: freeform but tagged by category:
- `[SETTINGS]` — permissions, allow/deny patterns
- `[AGENT]` — agent definition improvements
- `[WORKFLOW]` — session flow, delegation, review process
- `[TESTING]` — tester capabilities, Playwright, CLS
- `[BOOTSTRAP]` — bootstrap gaps, missing questions
- `[SECURITY]` — security patterns, auth, credential handling
- `[DOCS]` — documentation gaps

Example: `field-notes-jagdhuette-2026-04-02.md`

### 2. Contribute-Back Reports (`{project}-{date}.md`)

Formal reports generated by running `contribute-back.md` on a project after 10+ cycles. These are structured, categorized (transferable vs project-specific), and ready for user review + apply.

Example: `jagdhuette-full-2026-04-15.md`

## The flow

```
During sessions: dump field notes here as you discover things
                        ↓
After 10+ cycles: run contribute-back.md for a formal report
                        ↓
User reviews & marks APPROVED / DEFERRED / REJECTED
                        ↓
Run apply-contributions.md to implement approved items
                        ↓
Framework templates updated
```

Field notes feed into the contribute-back report. The contribute-back agent reads field notes as input alongside the project's agent definitions and CLAUDE.md.
